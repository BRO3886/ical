# Journal Entry 006 - 2026-02-26 - Agent skill flag documentation fixes

## Goal

Fix agent usability issues discovered during real-world use of the embedded `ical-cli` skill by an AI agent. The agent was failing to correctly use the `delete` and `update` commands due to missing/incorrect flag documentation in `SKILL.md`.

## What Changed

- **`skills/cal-cli/SKILL.md`**: Added delete example in Quick Start with `--force` flag
- **`skills/cal-cli/SKILL.md`**: Updated Row Numbers section to show both `ical delete 1 --force` and `ical delete 1` (with clarifying comment)
- **`skills/cal-cli/SKILL.md`**: Added prominent "Important for scripting" callout block distinguishing:
  - `delete` requires `--force` to skip confirmation (no `--confirm` flag exists)
  - `update` has no `--force` flag and no confirmation prompt at all
- Reinstalled skill via `ical skills install --agent claude`

## Key Insights

**Two separate agent failures observed**:

1. Agent tried `ical delete 11 --confirm` → exit code 1 (unknown flag). The agent guessed `--confirm` because the SKILL.md delete example showed `ical delete 1` with no flag and no mention of confirmation behavior. Fix: add explicit note that confirmation-skip flag is `--force`, not `--confirm`.

2. Agent tried `ical update 14 ... --force` → exit code 1. After seeing `delete` uses `--force`, the agent incorrectly assumed `update` also needed it. Fix: explicit callout that `update` has no `--force` flag and no confirmation at all.

**Root cause**: SKILL.md examples showed the "happy path" without documenting the interactive-vs-noninteractive distinction. Agents infer flag names from conventions and pattern-match from nearby commands — both failure modes were predictable.

**Pattern for agent-facing docs**: When a command has interactive confirmation, the non-interactive override flag MUST appear in the first example the agent will see (Quick Start / Row Numbers section), not buried in the reference table.

## Decisions Made

- Added a `> **Important for scripting**` blockquote to the Row Numbers section (high visibility, agents read top-down and this section is where they learn about row number usage)
- Kept the `ical delete 1` (no-force) example to show both forms exist, but added a comment and the `--force` form first so scripting agents default to the correct one
- Did not add `--force` to `update` (it doesn't exist and would break)

## Session 2 - 2026-02-26 - --id flag for show/update/delete + website docs

## Goal

Add an explicit `--id` flag to `show`, `update`, and `delete` commands so agents can look up events by full event ID without relying on positional args or prefix matching. Also update website docs and skill files to document the new flag.

## What Changed

- **`cmd/ical/commands/show.go`**: Added `showID string` var, `--id` flag, early mutual-exclusivity check, and `client.Event(showID)` branch before positional arg and picker paths
- **`cmd/ical/commands/delete.go`**: Same pattern — `deleteID`, `--id` flag, exclusivity check, exact lookup branch
- **`cmd/ical/commands/update.go`**: Same pattern — `updateID`, `--id` flag, exclusivity check, exact lookup branch
- **`skills/cal-cli/references/commands.md`**: Added `--id` to show/update/delete flag tables, updated event selection method lists, added mutual-exclusivity notes
- **`skills/cal-cli/SKILL.md`**: Added "Event ID flag (for scripts and agents)" subsection under Row Numbers with shell snippet showing `jq → --id` pattern
- **`website/content/docs/commands.md`**: Updated show/update/delete sections with `--id` flag tables, new examples, and scripting callout notes
- Reinstalled skill via `ical skills install --agent claude`

## Key Insights

**Implementation pattern** (identical across all three commands):
```go
idFlagSet := cmd.Flags().Changed("id")
if idFlagSet && len(args) > 0 {
    return fmt.Errorf("cannot use both --id and a positional argument")
}

if idFlagSet {
    event, err = client.Event(showID)  // exact lookup, no prefix search
    if err != nil {
        return fmt.Errorf("event not found: %w", err)
    }
} else if len(args) == 1 {
    event, err = findEventByPrefix(client, args[0])
    ...
} else {
    // interactive picker
}
```

Used `cmd.Flags().Changed("id")` instead of checking `showID != ""` — important distinction because empty string `--id ""` would be a valid (if broken) invocation otherwise.

**No prefix matching via `--id`**: Unlike positional args which fall through to `findEventByPrefix()` (row number → exact → prefix scan), `--id` goes straight to `client.Event()`. If not found, returns error immediately. This is intentional — agents should get a clear failure rather than a surprise prefix match.

**Agent usage pattern** documented in SKILL.md:
```bash
EVENT_ID=$(ical today -o json | jq -r '.[0].id')
ical show --id "$EVENT_ID"
ical update --id "$EVENT_ID" --title "New title"
ical delete --id "$EVENT_ID" --force
```

## Decisions Made

- `--id` is exact-only — no prefix fallback. Agents passing a full ID from JSON output should always get an exact match or a clear error, not a surprise ambiguous match.
- Mutual exclusivity is an error (not silently ignoring one) — fail fast rather than silent precedence rules that confuse agents.
- `--force` remains a separate flag on `delete` — `--id` is purely for event selection, orthogonal to confirmation behavior.
- Website docs updated in same session to keep all three doc surfaces (SKILL.md, references/commands.md, website) in sync.
